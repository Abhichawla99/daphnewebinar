## Section-by-Section Layout Guide

### HERO SECTION
**Goal:** Immediate emotional recognition + clear promise
- **Headline:** Direct statement that validates their current struggle
- **Subheadline:** Clarifies who this is for
- **Promise:** What they'll walk away with
- **Webinar Details Box:** Date, time in high-contrast container
- **Gift Callout:** FOMO element about exclusive attendee-only bonus
- **CTA:** Low-friction scroll button

**Emotional Arc:** "You're lost" → "Here's the map" → "Secure access now"

---

### PAIN SECTION
**Goal:** Deep resonance with current daily reality
- **Grid layout:** 8 pain points in cards
- **Each pain point:**
  - Short headline (the struggle)
  - Brief description (the lived experience)
- **No solution yet** - just acknowledgment
- **CTA:** "I Need This Structure"

**Emotional Arc:** "Someone finally gets it" → "This is exactly what I'm facing"

---

### FRAMEWORK SECTION
**Goal:** Shift from pain to possibility
- **Numbered pillars:** 1-5 with visual hierarchy
- **Each pillar includes:**
  - What it is (clear label)
  - What they'll learn (concrete outcome)
- **Language:** Specific, not theoretical
- **CTA:** "Register Now"

**Emotional Arc:** "Here's what's possible" → "This is doable" → "I can see myself doing this"

---

### ABOUT DAPHNE
**Goal:** Establish authority without ego
- **Photo/initial placeholder**
- **3-4 short paragraphs:**
  - Her experience with this exact audience
  - Her approach (practical, not inspirational)
  - Why she created this framework
- **No fluff, no credentials list** - just relevant expertise

**Emotional Arc:** "Can I trust this?" → "She knows this path" → "She's the guide I need"

---

### WHY THIS MATTERS
**Goal:** Contextualize the gap in their current care
- **Short, punchy paragraphs**
- **Builds the case:**
  - Doctors do medical treatment
  - No one does lifestyle structure
  - This fills that gap
- **Simple, direct language**
- **CTA:** "Get the Structure You Need"

**Emotional Arc:** "This explains why I feel so lost" → "This is the missing piece"

---

### URGENCY SECTION
**Goal:** Create legitimate scarcity
- **Dark background box** (visual contrast)
- **Bulleted reasons:**
  - Limited capacity
  - Black Friday offer (exclusive)
  - Replay access (for registrants only)
  - Gift disappears (FOMO)
  - Only public showing of 12-week system
- **CTA:** "I Don't Want to Miss This"

**Emotional Arc:** "I should decide" → "I need to decide now"

---

### SOCIAL PROOF
**Goal:** Normalize the transformation
- **6 testimonials** in grid
- **Each includes:**
  - Short quote (outcome-focused)
  - First name + last initial
  - Status (diagnosed/remission/treatment)
- **Placeholders provided** - replace with real testimonials when available

**Emotional Arc:** "People like me did this" → "They got results" → "I can too"

---

### FORM SECTION
**Goal:** Frictionless registration
- **Clean, centered layout**
- **Minimal fields:**
  - First name
  - Last name
  - Email
  - Phone (with country code selector)
- **Large, clear CTA button**
- **Thank you state** appears after submission
- **No distractions** - just the form

**Emotional Arc:** "This is easy" → "I'm registered" → "I'm in"

---

### FINAL CTA
**Goal:** Last chance to convert scrollers
- **High-contrast background**
- **Ultra-simple message:**
  - "Get clarity. Get structure."
  - "This is the plan you've been looking for."
- **Final CTA button**

**Emotional Arc:** "Last reminder" → "Now or never"

## Mobile-First Formatting Notes

### Typography Scale (Mobile):
- H1: 32px (hero headline)
- H2: 28px (section headlines)
- H3: 22px (framework items)
- Body: 16px
- Small: 14px

### Spacing:
- Section padding: 60px top/bottom
- Card spacing: 24px gap
- Form fields: 20px margin-bottom

### Touch Targets:
- All buttons: minimum 48px height
- Form inputs: 48px minimum
- CTAs: Full-width on mobile

### Grid Behavior:
- Pain grid: 1 column on mobile
- Framework: 1 column (stacked)
- Testimonials: 1 column below 768px
- Phone input: Stacked on mobile (country code above number)

## Technical Integration Notes

### Webhook Payload
The form sends this JSON structure:
```json
{
  "firstName": "string",
  "lastName": "string",
  "email": "string",
  "countryCode": "string",
  "phone": "string",
  "fullPhone": "string (countryCode + phone)",
  "registeredAt": "ISO 8601 timestamp"
}
```

### Country Code Options Included:
25 countries with flags and codes, defaulting to +1 (US/Canada)

### Form Validation:
- All fields required
- Email format validated (HTML5)
- Phone accepts numbers only
- Country code pre-validates format

### Success Flow:
1. Form submits to webhook
2. Response received
3. Form hidden
4. Thank you message displayed
5. Auto-scroll to thank you

### Error Handling:
- Network errors: Alert user to retry
- Server errors: Alert user to contact support
- All errors logged to console
